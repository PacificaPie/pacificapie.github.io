Two-Stage Generation — Rooting Out Garbled Chinese in AI Images 两段式生成 —— 根除 AI 生图中文乱码

The problem

Text-to-image models garble Chinese, reliably. The moment an event title enters the prompt, the model treats it as poster copy and paints the characters onto the image — mangled.

My call

This isn't a wording-tuning problem. It's a pipeline-structure problem: separate text from image. The AI should only ever paint a text-free background — the words belong to a different layer entirely.

What shipped

  • Two-stage generation — a text LLM first “translates” the event into a pure visual scene description, which is then fed to the text-to-image model.
  • Fallbacks by event type for when generation misbehaves.
  • Weakness-driven art direction — an impressionist brushwork style, chosen because it masks the AI's detail failures while staying on the brand palette.

Validation

Final images for all trending events landed in the table. When a batch rerun turned up anomalies, a “fallbacks-must-diverge” counter-evidence check localized the cause to a config that hadn't taken effect — not a flaw in the design.

What iterating taught me

The core of LLM engineering isn't prompt wording — it's pipeline structure.

问题

文生图模型渲染中文必乱码。事件标题一进 prompt，就被当成海报文字画到图上 —— 而且画坏。

我的判断

这不是措辞调优问题，是链路结构问题：图文分离。 AI 只画无文字背景图 —— 文字属于另一层。

交付

  • 两段式生成 —— 文本大模型先把事件 "翻译" 成纯画面场景描述， 再喂给文生图模型。
  • 按事件类型的兜底，应对生成异常。
  • "弱点驱动选型" 的画风 —— 选印象派笔触，因为它能掩盖 AI 细节翻车， 且贴品牌色板。

验证

全部热点事件成品图落表。批量重跑出现异常时，用 "兜底必分化" 的反证法把原因定位为 配置未生效，而非方案缺陷。

认知迭代

LLM 工程的核心不是 prompt 措辞，而是链路结构。
